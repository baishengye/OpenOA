# 新增一个可剥离复用模块（以 @itc/biometric 为样板）

所有端能力模块结构一致：**TS 统一 API + codegen spec + 三端原生 + 实现 `ItcModule` 契约**。下面以新建 `@itc/foo` 为例。

> 直接参考样板：[packages/biometric/](../packages/biometric/)。

---

## 1. 建包骨架

```
packages/foo/
├── package.json            # name=@itc/foo；codegenConfig；builder-bob
├── tsconfig.json           # 见下（不要 composite/references/paths-to-源码）
├── src/
│   ├── index.ts            # 对外统一 API + 实现 ItcModule（继承 BaseModule）
│   ├── NativeItcFoo.ts      # TurboModule spec（codegen 输入）
│   └── types.ts
├── android/                # Kotlin TurboModule + ReactPackage
│   ├── build.gradle
│   └── src/main/java/com/itc/foo/{ItcFooModule.kt,ItcFooPackage.kt}
├── ios/                    # ObjC++ TurboModule（.h/.mm）
├── harmony/biometric/...   # ArkTS RNOH TurboModule + RNPackage
└── ItcFoo.podspec
```

**关键 package.json 字段**（照抄 biometric，改名）：
```jsonc
{
  "name": "@itc/foo",
  "exports": { ".": { "source": "./src/index.ts", "react-native": "./src/index.ts",
    "import": { "types": "./lib/typescript/module/index.d.ts", "default": "./lib/module/index.js" } } },
  "peerDependencies": { "@itc/base": "workspace:*", "react": "*", "react-native": ">=0.77.0" },
  "react-native-builder-bob": { "source": "src", "output": "lib",
    "targets": [["module", {"esm": true}], ["typescript", {"esm": true}]] },
  "codegenConfig": { "name": "RNItcFooSpec", "type": "modules", "jsSrcsDir": "src",
    "android": { "javaPackageName": "com.itc.foo" } }
}
```

**tsconfig.json**：`extends ../../tsconfig.base.json`，只设 `rootDir/outDir/noEmit:false`。**不要** composite/references/`paths` 指向 base 源码（见 [TROUBLESHOOTING #4](踩坑速查.md)）。

## 2. codegen spec（src/NativeItcFoo.ts）

```ts
import type { TurboModule } from 'react-native';
import { TurboModuleRegistry } from 'react-native';

export interface Spec extends TurboModule {
  doSomething(arg: string, count: number): Promise<{ ok: boolean }>;
}
export default TurboModuleRegistry.getEnforcing<Spec>('ItcFoo');
```

铁律：**入参只用基本类型**（string/number/boolean），避免 iOS codegen 为对象入参生成 C++ 结构体。需要传对象时，由 `src/index.ts` 拆成位置参数。

---

## 3. 统一 API（src/index.ts，实现契约）

```ts
import { BaseModule, ItcError } from '@itc/base';
import NativeItcFoo from './NativeItcFoo';

class FooModule extends BaseModule {
  readonly name = 'foo';
  async isSupported() { try { /* 探测 */ return true; } catch { return false; } }
  protected async onInit() {}
  protected async onDestroy() {}
  async doSomething(arg: string, count: number) {
    try { return await NativeItcFoo.doSomething(arg, count); }
    catch (e) { throw ItcError.from(e, 'foo'); }
  }
}
export const foo = new FooModule();
```

错误统一 `throw ItcError.from(e, '<module>')`；事件经 `@itc/base` 的 `eventBus` 下发（推送/IM 类）。

---

## 4. 三端原生（签名对应 spec）

- **Android**：`ItcFooModule.kt extends NativeItcFooSpec`（codegen 生成的抽象类，名取自 spec 文件名）；`ItcFooPackage.kt extends BaseReactPackage`，`isTurboModule=true`。`build.gradle` 只声明 kotlin classpath（见 [TROUBLESHOOTING #10](踩坑速查.md)）。
- **iOS**：`ItcFoo.mm` 实现 `<NativeItcFooSpec>`（头文件 `<RNItcFooSpec/RNItcFooSpec.h>` 来自 codegenConfig.name）；`getTurboModule` 返回 `std::make_shared<NativeItcFooSpecJSI>(params)`；`RCT_EXPORT_MODULE()`。
- **鸿蒙**：`ItcFooTurboModule extends TurboModule`（`@rnoh/react-native-openharmony/ts`）；`ItcFooPackage extends RNPackage` 注册。

> Android 抽象类名 = `Native<SpecFileBaseName>Spec`；iOS 协议/JSI 名同理；JS 端 `getEnforcing('ItcFoo')` 的字符串 = 原生 `getName()` 返回值。三者要对齐。

## 6. 鸿蒙模块开发要点

鸿蒙端与 iOS/Android 的 codegen 体系不同，需要手动维护 C++ 桥接文件。以下是完整流程，以 `@itc/biometric` 为参考样板。

### 6.1 自包含目录结构

所有 C++ 和 ArkTS 产物放在模块包内，app 不生成任何模块级别的 cpp：

```
packages/foo/harmony/foo/src/main/
├── cpp/RNOH/generated/
│   ├── BaseItcFooPackage.h          # C++ Package 工厂（手写）
│   └── turbo_modules/
│       ├── ItcFoo.cpp               # TurboModule 桥（仿 codegen 产物手写）
│       └── ItcFoo.h
└── ets/
    ├── ItcFooPackage.ets            # ArkTS RNPackage（手写）
    └── ItcFooTurboModule.ets        # ArkTS TurboModule 实现（手写）
```

### 6.2 C++ 桥接文件写法

**ItcFoo.h**（仿 codegen 格式）：
```cpp
#pragma once
#include "RNOH/ArkTSTurboModule.h"

namespace rnoh {
class JSI_EXPORT ItcFoo : public ArkTSTurboModule {
  public:
    ItcFoo(const ArkTSTurboModule::Context ctx, const std::string name);
};
} // namespace rnoh
```

**ItcFoo.cpp**：
```cpp
#include "ItcFoo.h"

namespace rnoh {
using namespace facebook;

ItcFoo::ItcFoo(const ArkTSTurboModule::Context ctx, const std::string name)
    : ArkTSTurboModule(ctx, name) {
    methodMap_ = {
        ARK_ASYNC_METHOD_METADATA(doSomething, 2),  // 参数个数
        ARK_METHOD_METADATA(syncMethod, 1),          // 同步方法
    };
}
} // namespace rnoh
```

- `ARK_ASYNC_METHOD_METADATA` 对应返回 `Promise` 的方法
- `ARK_METHOD_METADATA` 对应同步/void 方法
- 参数个数按 JS spec 算（不含回调）

**BaseItcFooPackage.h**：
```cpp
#pragma once
#include "RNOH/Package.h"
#include "RNOH/ArkTSTurboModule.h"
#include "RNOH/generated/turbo_modules/ItcFoo.h"

namespace rnoh {
class BaseItcFooPackageTurboModuleFactoryDelegate : public TurboModuleFactoryDelegate {
  public:
    SharedTurboModule createTurboModule(Context ctx, const std::string &name) const override {
        if (name == "ItcFoo") return std::make_shared<ItcFoo>(ctx, name);
        return nullptr;
    };
};

class BaseItcFooPackage : public Package {
  public:
    BaseItcFooPackage(Package::Context ctx) : Package(ctx){};
    std::unique_ptr<TurboModuleFactoryDelegate> createTurboModuleFactoryDelegate() override {
        return std::make_unique<BaseItcFooPackageTurboModuleFactoryDelegate>();
    }
};
} // namespace rnoh
```

### 6.3 ArkTS TurboModule 实现

```typescript
// ItcFooTurboModule.ets
import { TurboModule, TurboModuleContext } from '@rnoh/react-native-openharmony/ts';

export class ItcFooTurboModule extends TurboModule {
  static readonly NAME = 'ItcFoo';

  constructor(ctx: TurboModuleContext) { super(ctx); }

  async doSomething(arg: string, count: number): Promise<{ ok: boolean }> {
    // 实现逻辑
    return { ok: true };
  }
}
```

```typescript
// ItcFooPackage.ets
import { RNPackage, TurboModulesFactory } from '@rnoh/react-native-openharmony/ts';
import { ItcFooTurboModule } from './ItcFooTurboModule';

class ItcFooTurboModulesFactory extends TurboModulesFactory {
  createTurboModule(name: string) {
    if (name === ItcFooTurboModule.NAME) return new ItcFooTurboModule(this.ctx);
    return null;
  }
  hasTurboModule(name: string) { return name === ItcFooTurboModule.NAME; }
}

export class ItcFooPackage extends RNPackage {
  createTurboModulesFactory(ctx: any) { return new ItcFooTurboModulesFactory(ctx); }
}
```

### 6.4 接入 app（三处修改）

**① `apps/oa/harmony/entry/src/main/cpp/CMakeLists.txt`** — 添加 glob：
```cmake
set(FOO_LIB_CPP "${CMAKE_CURRENT_SOURCE_DIR}/../../../../../../../packages/foo/harmony/foo/src/main/cpp")
file(GLOB FOO_LIB_CPP_FILES "${FOO_LIB_CPP}/RNOH/generated/turbo_modules/*.cpp")

add_library(rnoh_app SHARED
    ...
    ${FOO_LIB_CPP_FILES}
    ...
)
target_include_directories(rnoh_app PUBLIC ... "${FOO_LIB_CPP}")
```

**② `apps/oa/harmony/entry/src/main/cpp/PackageProvider.cpp`** — 注册：
```cpp
#include "RNOH/generated/BaseItcFooPackage.h"
// ...
return {
    std::make_shared<BaseItcFooPackage>(ctx),
    // ...
};
```

**③ `apps/oa/harmony/entry/src/main/ets/RNPackagesFactory.ets`** — ArkTS 侧注册：
```typescript
import { ItcFooPackage } from '@itc/foo/harmony/foo';  // 若 oh-package.json5 注册了 HAR
// 或者直接相对路径导入
export function createRNPackages(ctx: RNPackageContext): RNPackage[] {
  return [..., new ItcFooPackage(ctx)];
}
```

### 6.5 接入第三方 RNOH 库（HAR 形式）

部分三方库（如 CodePush、MMKV）以 `.har` 形式发布鸿蒙适配包。

**① oh-package.json5 声明 HAR 路径**（`apps/oa/harmony/entry/oh-package.json5`）：
```json5
"dependencies": {
  "@react-native-oh-tpl/react-native-foo": "file:../../node_modules/@react-native-oh-tpl/react-native-foo/harmony/foo.har"
}
```

**HAR 路径的 pnpm 宿主依赖陷阱**：`../../node_modules/` 解析到 `apps/oa/node_modules/`。如果该包只是 `packages/bar` 的依赖，pnpm 会把它装到 `packages/bar/node_modules/`，而不是 `apps/oa/node_modules/`——HAR 路径就会 404。
**解决**：在 `apps/oa/package.json` 的 `dependencies` 里也声明该包（即使 app 本身不直接用它）。

**② bridge 文件（仅在三方库要求 `@rnoh/react-native-openharmony/generated/ts` 时需要）**

某些三方 HAR 会 `import { TM } from '@rnoh/react-native-openharmony/generated/ts'`，但 RNOH 官方包不提供这个路径。需要手动创建 bridge 文件：

```
apps/oa/harmony/entry/oh_modules/@rnoh/react-native-openharmony/generated/ts.ts
```

**关键陷阱**：`entry/oh_modules/@rnoh/react-native-openharmony/` 是软链接，指向 `harmony/oh_modules/.ohpm/<hash>/oh_modules/@rnoh/react-native-openharmony/`。编译器用**真实路径**解析 import，相对路径从 `.ohpm` 深层出发根本到不了 app 目录。

**唯一可靠做法：把 spec 内联在 bridge 文件，不依赖任何外部 import**：

```typescript
// entry/oh_modules/@rnoh/react-native-openharmony/generated/ts.ts
export namespace RNC {}

export namespace TM {
  export namespace RTNFoo {
    export const NAME = 'RTNFoo' as const
    export interface Spec {
      doSomething(arg: string): Promise<object>;
      // ... 类型必须和 HAR 实现的实际返回类型匹配，
      //     codegen 生成的 Promise<void> 可能和实现不符，需手动核对
    }
  }
}
```

> ohpm install 后 oh_modules 会被清空，bridge 文件需重新创建。建议在项目 README 中标注。

## 7. 推送 / IM 的特别说明

- `@itc/push`：底层走第三方聚合（友盟/极光）+ iOS APNs + 鸿蒙 Push Kit；回调统一经 `eventBus.emit('push:message'|'push:opened')`。
- `@openim/rn-client-sdk-plus`：A/iOS 用官方 `open-im-sdk-rn`；**鸿蒙端需自编译 `openim-sdk-core`(Go) 为 OHOS arm64 `.so` + ArkTS NAPI 绑定**（最大风险项，建议独立 PoC）。
