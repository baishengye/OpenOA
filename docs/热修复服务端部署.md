# 自建 CodePush Server + 三端热更新接入实录

记录 OpenOA 自建 code-push-server(本地 dev)+ iOS/Android/鸿蒙三端接入热更新的完整流程。客户端 API 只暴露 `hotfix.*`(见 [@itc/hotfix](../packages/hotfix/)),本文档只讲服务端部署 + key 配置 + 发布。

> 三端客户端集成的坑见 [踩坑速查.md #11b/#19–22](踩坑速查.md) 和 [鸿蒙接入要点.md §8](鸿蒙接入要点.md)。

## 0. 关键认知(先看,避免踩坑)

> ⚠️ **server 选型血泪教训(2026-06 实测，务必先看)**：
> - **别用 `lisong/code-push-server` / `tablee/code-push-server:v0.5.2` 镜像** —— 太老，**不支持新版 `react-native-code-push`(8.x，依赖 `code-push@4.2.3` acquisition)** 请求的 `/v0.1/public/codepush/update_check` 路径，三端检查更新**全 404**。排查浪费极久。
> - ✅ **用 [shm-open/code-push-server](https://github.com/shm-open/code-push-server)**(react-native-code-push 官方推荐，镜像 `shmopen/code-push-server:latest`，支持 `/v0.1`)。
> - **client 传 snake_case 参数**：`deployment_key` / `app_version` / `client_unique_id`(**不是** camelCase)。自己 curl 测 server 必须用 snake_case，否则误判 404。正确测法：
>   ```bash
>   curl "http://<IP>:3000/v0.1/public/codepush/update_check?deployment_key=<真实Stagingkey>&app_version=1.0.0&client_unique_id=t"
>   # 通了返回 {"update_info":{"is_available":false,...}} + HTTP 200
>   ```
> - **国内 docker 拉镜像 `EOF`**：`~/.docker/daemon.json` 配 `registry-mirrors` 会被 Docker Desktop 重启时还原；改用**镜像名加前缀**绕过 —— docker-compose.yml 里写 `docker.1ms.run/shmopen/code-push-server:latest`、`docker.1ms.run/library/mysql:latest` 等([2026 可用加速器](https://github.com/dongyubin/DockerHub))。
> - **shm-open 用 curl 拿 token**(无需 cli)：`POST /auth/login {"account":"admin","password":"123456"}` → 返回 `{"results":{"tokens":"<jwt>"}}`，该 jwt 直接作 `Authorization: Bearer` 调 management API(建 app / 拿 key)。
> - 本次实际：server=`shmopen`，IP=`172.16.80.101`，三端已填各自 Staging key。
> - **REST API 发升级包(无需 cli，已验证)**：`POST /apps/<app>/deployments/Staging/release`，multipart `package=@包.zip` **必须显式 `;type=application/zip`**(否则被当 `application/octet-stream` 拒：`upload file type is invalidate`) + `packageInfo={"appVersion":"1.0.0",...}`。server 自己算 package_hash。成功返回 `{"msg":"succeed"}`，随后 update_check 即 `is_available:true` + `download_url`。
>   ```bash
>   curl -s -X POST -H "Authorization: Bearer $TOKEN" \
>     -F "package=@/tmp/cp.zip;type=application/zip" \
>     -F 'packageInfo={"appVersion":"1.0.0","isMandatory":false,"description":"v2"}' \
>     "$BASE/apps/OpenOA-ios/deployments/Staging/release"
>   ```
> - **`code-push-cli` 别装在 monorepo 工作区内**：shm-open cli 是 TS，`tsc` build 时会向上解析到 OpenOA 项目根的 pnpm `rimraf@6`(新 API 非 callable) → `TS2349 not callable`。装到**项目外**(如 `/Volumes/MacExtend/Code/ReactNative/Server/code-push-cli`)即避开。cli 仅用于打**真实** RN bundle 的 `release-react`；纯验证服务端流程用上面的 REST API 即可。

- **Access Token ≠ Deployment Key**：
  - **Access Token**：cli/REST API 登录认证用(web admin → Account/Access Key 拿),**不进客户端**。
  - **Deployment Key**：每个 app 的每个 deployment(Staging/Production)一个，**这才是填进客户端的**。
- **鸿蒙在原版 server 上建成 `os=Android`**：lisong 原版 server 只认 `iOS|Android|Windows`，无 harmony。但 **deployment key 与 os 无关**(只是标识)，鸿蒙 app 建成 Android、release/updateCheck 协议通用，照常工作。真遇到鸿蒙拉取异常再换 [react-native-oh-library fork](https://github.com/react-native-oh-library/code-push-server)。
- **dev 用 Staging，正式用 Production**。鸿蒙 `CodePushBuilder` 按 `isDebug` 自动选 Staging(debug)/Production(release)。
- **`DOWNLOAD_URL` 不能是 localhost**：真机下载更新包会拿到 localhost 失败，必须是本机局域网 IP。

## 1. 部署 server(本地 Docker，用完可整体清除)

用 [lisong/code-push-server](https://github.com/lisong/code-push-server) 原版(中文社区，docker 一键)。整套(MySQL+Redis+server)在 docker volume，`down -v` / 卸载 Docker Desktop 即清，不污染本机。

```bash
git clone https://github.com/lisong/code-push-server.git
cd code-push-server/docker
# 改 docker-compose.yml：
#   DOWNLOAD_URL → http://<本机局域网IP>:3000/download   （本项目 = http://172.16.80.141:3000/download）
#   TOKEN_SECRET → 随机串
docker-compose up -d
docker-compose ps                 # 容器 Up
curl -I http://localhost:3000/    # 200
```

- web 管理：浏览器 `http://localhost:3000`，默认 `admin / 123456`(**改密码**)。
- 本机局域网 IP：`ipconfig getifaddr en0`(本项目当前 = `172.16.80.141`)。
- 清除：`docker-compose down -v`。

## 2. 拿 Access Token

web admin 登录 → Account/Access Key 页面生成(或 cli login 流程)。本次拿到的 token 用于下一步 REST API 认证。

## 3. 建三端 app + 拿 Deployment Key（REST API，无需装 cli）

server 自带 web 界面但无 cli 二进制；直接用 access token 调 REST API：

```bash
TOKEN=<你的 access token>
BASE=http://localhost:3000

# 建 app（鸿蒙用 Android 替代）
for pair in "OpenOA-ios:iOS" "OpenOA-android:Android" "OpenOA-harmony:Android"; do
  name="${pair%%:*}"; os="${pair##*:}"
  curl -s -X POST -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
    -d "{\"name\":\"$name\",\"os\":\"$os\",\"platform\":\"React-Native\"}" "$BASE/apps"
done

# 拿每个 app 的 Staging/Production deployment key
for name in OpenOA-ios OpenOA-android OpenOA-harmony; do
  curl -s -H "Authorization: Bearer $TOKEN" "$BASE/apps/$name/deployments"
done
```

> 建 app 自动创建 Staging + Production 两个 deployment，各带 key。返回的 `key` 字段即 deployment key。

## 4. 三端填 Deployment Key + ServerUrl

dev 用 **Staging** key，serverUrl 统一 = `http://<本机局域网IP>:3000`(本项目 `http://172.16.80.141:3000`)：

| 端 | 文件 | DeploymentKey 字段 | ServerUrl 字段 |
|---|---|---|---|
| iOS | [ios/OpenOA/Info.plist](../apps/oa/ios/OpenOA/Info.plist) | `CodePushDeploymentKey` | `CodePushServerURL` |
| Android | [android/app/src/main/res/values/strings.xml](../apps/oa/android/app/src/main/res/values/strings.xml) | `CodePushDeploymentKey`(空也必须存在，否则 autolink PackageList 编译失败) | `CodePushServerUrl` |
| 鸿蒙 | [harmony/entry/build-profile.json5](../apps/oa/harmony/entry/build-profile.json5) `buildProfileFields` | `Staging`/`Production` | `ServerUrl` |

- iOS HTTP 明文：`Info.plist` 的 `NSAllowsLocalNetworking=true` 已放行私有段 IP(172.16.x.x)，无需额外 ATS 配置。
- 鸿蒙 `Staging`+`Production` 都填(debug 走 Staging，release 走 Production)；[EntryAbility.onCreate](../apps/oa/harmony/entry/src/main/ets/entryability/EntryAbility.ets) 里 `import BuildProfile from 'BuildProfile'` 读出、`initHotfix({ staging: BuildProfile.Staging, … })` 注入 `AppStorage`。**不能 `initHotfix()` 无参**（否则写空串、白填）；改了 `buildProfileFields` 要 **DevEco Clean 后重新 build**（BuildProfile.ets 才重生成）。验证：运行日志 `checkForUpdate-config` 里能看到实际的 deploymentKey/serverUrl。

## 5. 验证连通(先不发包)

三端重新 run → 进「热更新」tab 点**检查更新** → 从 `Network request failed` 变成 **up-to-date(无更新)**。**这一步成功 = 整个链路打通**。

## 6. 发布更新(release，阶段二)

需要 [code-push-cli](https://github.com/react-native-oh-library/code-push-cli)(clone + `npm install`，本地跑不污染全局)，或 web 上传：

```bash
# iOS / Android：cli 自动打 jsbundle 并上传
code-push release-react OpenOA-ios     ios
code-push release-react OpenOA-android android

# 鸿蒙：先打 bundle.harmony.js，再 release 上传
cd apps/oa && npx react-native bundle-harmony --dev false --config metro.config.harmony.js
code-push release OpenOA-harmony <bundle.harmony.js 路径> "<appVersion>" -d Staging
```

客户端下次 check 即可拉取、按 installMode 生效。

## 7. 清除(预研结束)

```bash
cd code-push-server/docker && docker-compose down -v   # 清容器+数据
```
或直接卸载 Docker Desktop.app —— 本机零残留。客户端的 key/serverUrl 是项目配置，按需保留或清空。
